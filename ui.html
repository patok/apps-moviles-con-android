Interfaz de Usuario (UI)

View y ViewGroup

Todos los elementos de la interface de usuario de una app se construyen utilizando View y ViewGroup.
Una View es un objeto que dibuja algo en la pantalla y que permite al usuario interactuar con él. Un ViewGroup es un objecto que contiene otros objeto Views (u otros ViewGroup) y define la disposición de la interface.

UI Layout

Usando una jerarquía de objetos View y ViewGroup se construye la interface. Cada grupo de views es un contenedor invisible que organiza las view subyacentes o hijas, las cuales pueden ser controles de entrada u otro tipo de componentes que dibujan alguna parte de la interface.

UI Layout

Android provee una colección de subclases View y ViewGroup que constituyen los controles de entrada (botones, campos de edición, etc) y varios modelos de disposiciones o layouts (LinearLayout, RelativeLayout).

Para definir el layout típicamente se utiliza XML: 


<?xml version="1.0" encoding="utf-8"?>
<LinearLayout xmlns:android="http://schemas.android.com/apk/res/android"
              android:layout_width="fill_parent" 
              android:layout_height="fill_parent"
              android:orientation="vertical" >
    <TextView android:id="@+id/text"
              android:layout_width="wrap_content"
              android:layout_height="wrap_content"
              android:text="I am a TextView" />
    <Button android:id="@+id/button"
            android:layout_width="wrap_content"
            android:layout_height="wrap_content"
            android:text="I am a Button" />
</LinearLayout>

Views: operaciones

  • setear VISIBILITY: si está visible o no?
  • setear estado checked
  • definir Listeners: código para manejar eventos sobre la vista (OnClick, OnFocusChanged, etc)
  • setear propiedades: opacidad, background, rotación
  • gestionar foco

Views: componentes ejemplos

  • widgets de propósito general: Button, TextView, EditText, ListView, CheckBox, RadioButton, Gallery, Spinner
  • widgets especializados: AutoCompleteTextView, ImageSwitcher, TextSwitcher
  • layouts: LinearLayout, FrameLayout, RelativeLayout

 

siempre es posible extender las clases de View para proveer tu propio elemento (ver custom components)

AdapterView y Adapter

Hay componentes diseñados específicamente para mostrar distintos tipos de datos según la necesidad del desarrollador, son distintas subclases de un ViewGroup de nombre AdapterView, ej. ListView, Spinner.
Las vistas hijas y sus datos respectivos son gestionados por la View sino por otra clase llamada un Adapter, ej.: ListAdapter, SpinnerAdapter y ArrayAdapter

Layouts

Distintos tipos de Layouts proveen distintas estrategias para armado de la interface visual: LinearLayout, FrameLayout, RelativeLayout.

Todos incorporan parametrización común. Los mas importantes son los relativos a las dimensiones: android:layout_width and android:layout_height, para cada dimensión se puede especificar:

  • MATCH_PARENT - la vista trata de agrandarse tanto como el padre lo permita
  • WRAP_CONTENT - la vista adquiere el tamaño suficiente para su contenido
  • número exacto - ej. 140 dip (density independent pixel)
