La plataforma Android

  • Android es un sistema operativo para móviles adquirida por Google Inc.
  • basado en una versión modificada del kernel de Linux
  • desarrollado en conjunto con el OHA
  • Opera con distintos componentes a distintos niveles: kernel SO, librerías de sistema, framework de aplicaciones, y aplicaciones principales
  • provee herramientas para el desarrollo de aplicaciones: SDK (soft development kit), que incluye librerías y herramientas de desarrollo.

La arquitectura de Android

Linux kernel

  • Basado en el Linux kernel, pero NO es Linux
  • No tiene sistema de ventanas nativo
  • No tiene soporte de glibc
  • No incluye las utilidades estándard de Linux (apt)

Linux kernel

Porqué el Linux kernel?

  • Seguridad - arquitectura de permisos (aisla datos y recursos entre apps)
  • Gestor de memoria y procesos (evita que procesos interfieran entre si)
  • Entrada/Salida de Red y sistema de ficheros

Linux kernel

Porqué el Linux kernel? (cont)

  • Manejador de dispositivos (Device Drivers) y HAL - componentes de hardware: LCDs, touch-screens, cámaras, sensores, sdcards
  • Librerías compartidas en forma de módulos enchufables(plugable)
  • Código abierto y comunidad muy activa

Linux kernel

Mejoras al kernel (específicas Android)

  • Alarma - permite despertar dispositivo en base momento de tiempo o intervalo específico
  • memoria compartida (Ashmem) -
  • Binder p/ IPC (comunicación inter procesos via ashmem)
  • Gestión Energía (Power Management) - politicas + agresivas p/ uso de batería (wake locks)
  • Low memory killer
  • Kernel debugger (ADB)
  • Logger

Librerías Nativas

  • Bionic(libC): creación procesos e hilos, alocación de memoria, funciones/rutinas matemáticas
  • Librería de funciones
  • Servicios nativos
  • HAL - librerías acceso hardware

Librerías Nativas

Bionic Libc

  • Libc implementación optimizada
  • Licencia: GPL vs BSD - drivers propietarios cerrados
  • Tamaño: c/proceso incluye copia
  • Velocidad: muy rápida p/ manejo hilos
  • POSIX : características limitadas
  • NO compatible con GNU Libc
  • Se provee NDK, compila a Bionic

Librerías Nativas

Principales funciones

  • WebKit motor web
  • Media Framework: video, audio, codecs
  • SQLite: base de datos relacional embebida

Servicios Nativos

Principales controladores de I/O de la plataforma

  • Surface Flinger: display render/composer
  • +Surface Flinger: 2D/3D, OpenGL, hardware accel, double-buffering
  • Audio flinger: mezclador de audio
  • HAL: abstracciones intermedias entre lib nativas y drivers kernel

Android Runtime

Permite la ejecución de aplicaciones Android

  • Core Java Libraries
  • Dalvik Virtual Machine (hoy ART)

Core Java Libraries

Típicamente escritas en lenguaje Java, provee un gran número de clases reutilizables

  • java classes básicas: java.*, javax.*
  • ciclo de vide del app: android.*
  • internet/servicios web: org.*
  • tests de unidad: junit.*

      • ref: https://developer.android.com/reference/packages

Dalvik VM

  • escribe la aplicación en lenguaje Java
  • es compilado a un .class (bytecode standard)
  • una herramienta llamada DX transforma el bytecode Java, en un fichero ùnico con formato llamado DEX
  • fichero DEX file es empaquetado con los resources de la app e instalado en dispositivo
  • el usuario ejecuta la app, la DVM entonces ejecuta las clases del DEX file

      • Esto es así porque la DVM fué específicamente diseñada para ejecutarse en dispositivos de recursos reducidos: CPU mas lentas, menos memoria, batería limitada.

Application Framework

Software reusable que las aplicaciones móviles requieren (Java bindings)

  • Activity Manager: ciclo de vida de las apps(Activities)
  • Package Manager: archivos de aplicaciones instaladas (*.apk)
  • Window Manager: se ocupa de gestionar ventanas de apps

Application Framework (cont)

  • Resource Manager: maneja recursos de las app - layouts, graficos, texto localizado, audio
  • Content Providers: base de datos que permite a las apps almacenar y compartir datos
  • Notification Manager: se encarga la barra de notificaciones (eventos desde las apps)
